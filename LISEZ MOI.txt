Pour faire marcher l'interface web, il faut éditer le fichier config.php et modifier les paramètres en fonction de vos besoins.

Dans la base de données de VT, il faut créer une table "login_prof". Le script de création est en bas de cette page. Ensuite, il faut lancer le script "generateur_login_prof.php". Pour les profs, le login perso permet de mettre des rendez-vous perso et d'avoir l'interface "mes modules", "Mes heures" et Ma config"... Le login générique ne le permet pas. Par défaut, le login créé est de la forme suivante : 7 premières lettres du nom suivies de 1 (ou 2 ou 3... s'il y a des homonymes). Le mot de passe est le nom. 
Ensuite, pour le compte qui sera administrateur, il faut mettre le champ "admin" à 1 dans la table "login_prof". Ainsi, l'administrateur pourra modifier les droits de tous les utilisateur depuis l'interface web. Le droit "Export vers giseh" sert a générer un bilan des heures afin de les importer dans le logiciel de l'université Paris 10. Si vous n'avez pas le logiciel Giseh, il faut pas activer ce droit pour vos utilisateurs.

Pour créer les logins des étudiants, il faut lancer le script "generateur_login_etudiant.php".Par défaut, le login créé est de la forme suivante : première lettre du prénom suivie des 6 premières lettres du nom suivies de 1 (ou 2 ou 3... s'il y a des homonymes). Sinon, on peut le faire depuis VT en renseignant le champ "identifiant" dans "ajouter modifier détruire étudiant". Attention, l'étudiant doit forcement être associé à au moins un groupe pour que ça marche.

Pour pouvoir faire un pré-tri au niveau des groupes dans l'interface web des profs, il faut associer un "niveau" à chaque groupe dans VT. Il faut tout d'abord créer des niveaux dans "filière/niveaux" puis associer ces niveaux à chaque groupe dans "groupes/ajouter modifier détruire".

Pour lancer la génération des fichiers ics de manière régulière, il faut utiliser le service proposé par http://www.onlinecronjobs.com pour lancer les scripts qui sont dans les répertoires "icsetudiant", "icsprof" et "icssalle".

Dans la base de données de VT, il faut créer une table "compteur". Dans cette table, il faut créer le champ "id_compteur" en Int(2) puis créer le champ "valeur" en int(11) et avec comme valeur par défaut zéro. Le script de création est en bas de cette page.

Il faut créer une table "vacances" avec un champ "date" de type "date" et rentrer les jours des vacances scolaire et les jours fériés au format aaaa-mm-jj.

Pour que l'affichage du nom des séances marche correctement, il faut que les enseignements aient un nom qui respecte le format suivant : xxxx_nom_xx_xxx. En effet, pour connaitre le nom de l'enseignement, le script regarde ce qu'il y a entre le premier _ et le deuxième _.
Par exemple, pour un enseignement de TD de mécanique des fluides dans la filière GMP, voici ce que ça donne : GMP_MECANIQUE DES FLUIDES_TD_GROUPE 1
 et "MECANIQUE DES FLUIDES" s'affichera dans la case correspondant à la séance.

La couleur des séances correspond à la couleur des groupes que vous avez définie dans VT.

Les réservations placées par les profs ont pour code proprietaire 999. Si vous souhaitez les modifier depuis VT, il vous faudra créer un utilisateur avec le code propriétaire 999.

Si l'interface web affiche des messages d'erreurs comme quoi certaines variables ne sont pas définies..., ce n'est pas grave. Il suffit d'ajouter dans le répertoire un fichier .htaccess avec à l'intérieur la ligne "php_flag display_errors off" pour les cacher. J'ai normalement corrigé toutes les erreurs et aucune ne devrait s'afficher mais si vous en trouvez une, merci de me la signaler.
